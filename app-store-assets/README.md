# App Storeスクリーンショット素材

## 元画像の配置

iPhoneで撮影した未加工のPNGを、次の名前で `source` フォルダへ配置します。

1. `01-photo.png` — 写真モードの通常画面
2. `02-video.png` — 動画モードの通常画面
3. `03-ratio.png` — 画角変更画面
4. `04-controls.png` — 明るさ・ズーム調整画面
5. `05-save-share.png` — 撮影後の保存・共有画面

赤枠、書き込み、通知、個人情報が入った画像は使用しません。

## 推奨する被写体と撮影構成

全体を「窓辺の静かな時間」という一つのテーマで統一します。明るい窓の近くに、観葉植物、無地の白いマグカップ、表紙に文字のないノートを置きます。木目または無地のテーブルを使い、企業ロゴ、住所、手書きメモ、PC画面、人物の顔は写しません。

| 元画像 | 推奨する画面 |
|---|---|
| `01-photo.png` | 写真モード。観葉植物と白いマグを三分割構図で写す。最も明るく整ったメイン画像にする。 |
| `02-video.png` | 動画モード。透明なグラスへ水を注ぐ場面、または植物の葉が揺れる場面を写す。人物の顔は入れない。 |
| `03-ratio.png` | `1:1`画角。マグ、ノート、植物を中央にまとめ、正方形の構図が分かる状態にする。 |
| `04-controls.png` | 植物の葉または花へ寄り、明るさとズームのスライダーが見える状態にする。 |
| `05-save-share.png` | 1枚目で撮影した完成写真の確認画面を表示し、保存・共有操作が分かる状態にする。 |

撮影前に机上を整理し、自然光を横から当てます。TestFlightへの戻る表示、通知、低バッテリー警告などが消えてから撮影します。5枚で被写体と色調を揃えると、App Store上で一つのシリーズとして見えます。

## 生成

リポジトリ直下で次を実行します。

```powershell
node scripts/generate-app-store-screenshots.js
```

`output` にApp Store用の縦画像（1320 × 2868 PNG）が生成されます。

## 掲載コピー

| # | コピー |
|---|---|
| 1 | マナーモードで／静かに撮影 |
| 2 | 写真も動画も／これひとつ |
| 3 | シーンに合わせて／画角を選択 |
| 4 | 明るさとズームを／直感的に調整 |
| 5 | 撮ったらすぐに／保存・共有 |

デザイン方針は [DESIGN_PHILOSOPHY.md](DESIGN_PHILOSOPHY.md) を参照します。
